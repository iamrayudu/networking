backend/skills/ms_search_query.txt
========================
MICRO-SKILL: BUILD LINKEDIN SEARCH QUERY
You are building a LinkedIn people search query for one company and role.
Your output is used directly as a search URL parameter. Be precise.

RULES:
- Use the company's exact LinkedIn name — not a shortened version
- Filter: 2nd degree connections first. If zero results: widen to 3rd.
- People filter only — no jobs, no companies, no posts
- Do not include seniority filter in the URL — LinkedIn's seniority filter is unreliable
- Target titles to search for (in priority order):
    Engineering Manager, Director of Engineering, VP Engineering,
    Head of Product, Director of Product, Senior PM, Principal PM,
    Staff Engineer, Senior Engineer, Data Engineering, ML Engineer,
    Platform Engineer, Automation Engineer
- Build 2 query variants: one broad (company name only), one targeted (company + role keyword)
- Return JSON: {"query_broad": "...", "query_targeted": "...", "company_linkedin_name": "..."}

WHAT NOT TO DO:
- Do not search for Recruiter, Talent, HR, People Partner, Sourcer — ever
- Do not use the job title from the job description verbatim — it may not match LinkedIn titles
- Do not add location filter unless the role is explicitly location-specific

backend/skills/ms_profile_extract.txt
========================
MICRO-SKILL: EXTRACT DATA FROM A LINKEDIN PROFILE PAGE
You have just loaded a LinkedIn profile page via Selenium.
Your job: extract exactly these fields and nothing else.
Do not summarise. Do not interpret. Extract what is there.

FIELDS TO EXTRACT (use null if not found — never guess):
  name:               full name from the page header
  title:              current job title — exact text, no paraphrasing
  company:            current company name — exact text
  linkedin_url:       current browser URL
  headline:           the short tagline under their name
  summary:            first 400 characters of the About section — empty string if none
  current_role_duration: time in current role — e.g. "2 yr 3 mo" — null if not shown
  past_roles:         list of up to 3 previous job titles + companies
  mutual_connections: integer — 0 if none
  connection_degree:  "1st" / "2nd" / "3rd"
  recent_activity:    topic of most recent post or comment — null if none visible
  skills_visible:     list of up to 5 skills — empty list if hidden

QUALITY RULES:
- If page did not load: return null for all fields, set {"load_failed": true, "reason": "..."}
- If profile is in another language: extract anyway, note the language
- Never combine fields
- current_role_duration must be string like "1 yr" or "6 mo" — not a date range

========================
MICRO-SKILL: SCORE A LINKEDIN PROFILE FOR SUDHEER
You have a LinkedIn profile and Sudheer's full context.
Score this person 1-10 for how worthwhile it is to reach out.

SCORING RUBRIC:
  9-10: Title is exactly right (EM, Director, VP, Senior PM) AND 2nd degree
        AND recent activity AND technical/AI/product signal
  7-8:  Good title match, 2nd or 3rd degree, some technical signal
  5-6:  Adjacent title OR 3rd degree no mutuals OR thin profile
  1-4:  Any of: Recruiter/HR/Talent in title, no relevance, 3rd degree no signal

HARD RULES:
- Never score a Recruiter above 4 regardless of anything
- 2nd degree mediocre title beats 3rd degree perfect title
- Recent activity is a strong positive signal

RETURN JSON:
  score:       integer 1-10
  reason:      2 sentences max — WHY, name specific signals
  fit_angle:   strongest fit: "AI/automation builder" / "full-stack enterprise" /
               "data+analytics" / "certifications" / "product thinking"
  hook_source: "mutual_connection" / "recent_activity" / "role_signal" /
               "ai_signal" / "company_direction" / "past_role"
  surface:     true if score >= 6, false if score < 6

MICRO-SKILL: DRAFT A LINKEDIN CONNECTION MESSAGE FOR SUDHEER
You have the full context object. Write two draft messages.
One message. Done right. That is the entire job.

VOICE — Sudheer is a builder. Direct. No filler. No corporate language.

  "I hope this finds you well", "I'd love to", "I came across your profile",
  "I was really impressed", "I'm very passionate", "excited to connect",
  "I'd be honored", "looking forward to hearing from you",
  "I noticed we share similar interests", "exploring new opportunities",
  "open to new challenges", "results-driven", "proven track record",
  "leverage" (as business verb), "synergy", "circle back", "touch base"

  VP / Director / C-Suite:    120-180 characters
  Senior Manager / Head of:   180-240 characters
  Senior IC / Staff:          200-260 characters
  Manager / PM / Engineer:    200-280 characters
  Hard ceiling: 300 characters (LinkedIn limit)

STRUCTURE — 3 parts, must not sound structural:
  LINE 1 — hook: one specific real observation about THEM
    Use hook_source: mutual_connection / recent_activity / ai_signal / role_signal
  LINE 2 — connection: one sentence naming what Sudheer builds
    Use fit_angle: AI/automation builder / full-stack enterprise / data+analytics
  LINE 3 — close (optional, max 7 words):
    "Worth connecting." or "Wanted to be in your network."

PERSONALIZATION CHECK before finalising:
  [ ] Line 1 references something specific to THIS person (not just title)
  [ ] Line 2 names something Sudheer actually builds
  [ ] Zero banned phrases
  [ ] Under target length for this person's seniority
  [ ] Reads like one person to another — not a template

RETURN JSON:
  draft_a, draft_b, recommended, hook_source,
  char_count_a, char_count_b, applied_preferences (list)
