DIRECTIVE 04 — PRE-ACTION CHECKLIST FOR CHROME ACTIONS
========================================================
Problem being solved: the LinkedIn agent clicks and navigates without
first verifying the page loaded correctly, the expected element exists,
and the result makes sense. A failed page load silently produces garbage
data. A CAPTCHA gets treated as a profile. An empty search result gets
processed as if it had hits.

Fix: a pre-action guard function that runs before every Selenium action
that matters. If the guard fails: the action does not run. The agent
logs it, emits the right event, and decides what to do next.

Do NOT rebuild linkedin_agent.py. Add the guard layer on top.

  ── GUARD 1 ──
  def guard_page_loaded(driver, expected_url_fragment: str = None,
                        timeout: int = 8) -> dict:
      """
      Checks the page loaded successfully before any extraction.
      Returns: {"ok": bool, "reason": str}

      Checks in order:
        1. driver.current_url does not contain "linkedin.com/login"
           If it does: {"ok": False, "reason": "redirected_to_login"}
        2. If expected_url_fragment provided: it must be in driver.current_url
           If not: {"ok": False, "reason": "wrong_page"}
        3. document.readyState == "complete" via driver.execute_script()
           If not complete after timeout: {"ok": False, "reason": "page_not_ready"}
        4. Check for CAPTCHA: look for element with id "captcha-challenge"
           or text "Let's do a quick security check" anywhere on page
           If found: {"ok": False, "reason": "captcha_detected"}
        5. Check for "Page not found" or "This profile is no longer available"
           If found: {"ok": False, "reason": "profile_not_found"}
        6. All checks pass: {"ok": True, "reason": "ok"}
      """

  ── GUARD 2 ──
  def guard_search_has_results(driver) -> dict:
      """
      Checks a LinkedIn people search returned at least one result.
      Returns: {"ok": bool, "count": int, "reason": str}

      Checks:
        1. Look for element class "search-results-container" — must exist
           If not: {"ok": False, "count": 0, "reason": "no_results_container"}
        2. Look for "No results found" text anywhere on page
           If found: {"ok": False, "count": 0, "reason": "zero_results"}
        3. Count profile card elements (class "entity-result__title-text")
           If count == 0: {"ok": False, "count": 0, "reason": "zero_profiles"}
        4. If count > 0: {"ok": True, "count": count, "reason": "ok"}
      """

  ── GUARD 3 ──
  def guard_profile_is_readable(driver) -> dict:
      """
      Checks a profile page has enough data to be worth extracting.
      Returns: {"ok": bool, "reason": str, "quality": str}

      Checks:
        1. guard_page_loaded() must pass first — if not: return its result
        2. Profile name element must exist (h1 in main content area)
           If not: {"ok": False, "reason": "no_name_element", "quality": "empty"}
        3. Check if profile is private or restricted:
           Look for "LinkedIn Member" as the name text
           If found: {"ok": False, "reason": "private_profile", "quality": "private"}
        4. Check content richness:
           - Has About section: True/False
           - Has Experience section with at least 1 entry: True/False
           - Has headline text: True/False
           Score: 0 = only name visible, 1 = name + title, 2 = name + title + one section,
                  3 = name + title + about + experience
           If score == 0: {"ok": False, "reason": "profile_too_sparse", "quality": "empty"}
           If score >= 1: {"ok": True, "reason": "ok", "quality": "good" if score >= 2 else "minimal"}
      """

  ── GUARD 4 ──
  def guard_connect_button_available(driver) -> dict:
      """
      Checks the Connect button is available on a profile page before clicking.
      Returns: {"ok": bool, "reason": str, "button_type": str}

      Checks:
        1. guard_page_loaded(driver, "linkedin.com/in/") must pass
        2. Look for primary Connect button (aria-label contains "Connect")
           If found: {"ok": True, "reason": "ok", "button_type": "direct"}
        3. If not found: look for "More" button that might contain Connect
           Click More, check dropdown for Connect option
           If found: {"ok": True, "reason": "ok", "button_type": "in_more_menu"}
        4. Check if already connected (button says "Message" or "Following")
           If so: {"ok": False, "reason": "already_connected", "button_type": "none"}
        5. Check if pending (button says "Pending")
           If so: {"ok": False, "reason": "invitation_pending", "button_type": "none"}
        6. Nothing found: {"ok": False, "reason": "connect_unavailable", "button_type": "none"}
      """

  ── GUARD 5 ──
  def guard_message_box_open(driver) -> dict:
      """
      Checks the "Add a note" message box is open and ready to type.
      Called after clicking Connect and then "Add a note".
      Returns: {"ok": bool, "reason": str}

      Checks:
        1. Look for textarea with placeholder "Add a note..." or similar
           If found and visible: {"ok": True, "reason": "ok"}
        2. If not found: look for "Add a note" button and click it
           Wait 2 seconds, check again for textarea
           If found after click: {"ok": True, "reason": "ok"}
        3. If still not found: {"ok": False, "reason": "message_box_not_found"}
        4. Check textarea character limit warning not already showing
           (means box is empty and ready)
           If already showing: {"ok": False, "reason": "textarea_not_empty"}
      """
