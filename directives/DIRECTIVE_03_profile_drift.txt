DIRECTIVE 03 — FIX PROFILE DRIFT
==================================
Problem being solved: Sudheer's profile is loaded once at context build,
but by step 4 or 5 the agent's Claude calls have drifted away from his voice.
The tone_guide gets diluted because it sits deep in a long prompt and Claude
deprioritises it. Messages stop sounding like Sudheer and start sounding generic.

Fix: re-anchor the voice at every Claude call that produces user-facing text.
Not by reloading the full profile — by injecting a tight VOICE ANCHOR
(10-15 lines) as the FIRST thing in every system prompt that writes words
Sudheer will send or review.

Do NOT rebuild anything. Surgical additions only.

CHANGE 1 — CREATE NEW FILE

File to create: backend/memory/voice_anchor.py

Tell Claude Code:
  Create this file. It builds a short, sharp voice anchor string
  from Sudheer's tone_guide and learned preferences.
  This string gets prepended to every system prompt that writes
  text Sudheer will send or review.

  ANCHOR_INTRO = "SUDHEER'S VOICE — READ THIS FIRST:"

  def build_voice_anchor(ctx: dict) -> str:
      """
      Builds a 10-15 line voice anchor from ctx.
      Pulls from ctx["permanent"]["tone_guide"] and ctx["permanent"]["preferences"].

      Structure of the anchor — hardcode this structure, fill from ctx:

      SUDHEER'S VOICE — READ THIS FIRST:
      He is a full-stack developer and AI/automation builder. Not a job seeker. A builder.
      He writes like he talks: direct, specific, no filler.
      He never says: [top 5 banned phrases extracted from tone_guide]
      He sounds like: [one example sentence from tone_guide if present, else omit]
      His current strongest angle: [fit_angle from ctx["current_score"] if set, else omit]
      Active preferences this session:
      [top 3 rules from ctx["permanent"]["preferences"] — one line each]

      Rules for extraction:
      - banned phrases: find lines starting with "-" under "WHAT I NEVER SAY" in tone_guide
        Take the first 5. Format as comma-separated inline list.
      - example sentence: find first line under "Example of an opening I love" in tone_guide
        If not found: omit the line entirely.
      - preferences: split preference_summary by newline, take first 3 non-empty lines.
        If no preferences yet: omit the preferences section entirely.
      - fit_angle: only include if ctx["current_score"]["fit_angle"] is not None.

      Keep the anchor under 200 words. If it exceeds 200 words: trim preferences to 2,
      trim banned phrases to 3. Voice anchor must be SHORT — it is repeated often.

      Returns: formatted string
      """

  def prepend_voice_anchor(ctx: dict, system_prompt: str) -> str:
      """
      Prepends the voice anchor to an existing system prompt.
      Adds a blank line between anchor and the rest of the prompt.

      Format:
        {voice_anchor}

        {original system_prompt}

      Returns: combined string
      """

CHANGE 4 — ADD DRIFT DETECTION
═══════════════════════════════════════════════
File to modify: backend/agents/message_agent.py

Tell Claude Code:
  Add a post-draft check function. Call it immediately after draft_message()
  returns. If the draft fails the check, regenerate once with an explicit
  correction instruction. Do not regenerate more than once.

  BANNED_PHRASES = [
      "i hope this finds you",
      "i hope this message finds you",
      "i'd love to",
      "i would love to",
      "i came across your profile",
      "i was impressed",
      "i'm passionate",
      "excited to connect",
      "i'd be honored",
      "looking forward to hearing",
      "exploring new opportunities",
      "open to new challenges",
      "results-driven",
      "proven track record",
  ]

  def check_voice_drift(draft: str) -> list[str]:
      """
      Checks a message draft for banned phrases.
      Returns list of banned phrases found (empty list = pass).
      Case-insensitive check.
      """

  def fix_drift(draft: str, violations: list[str], ctx: dict) -> str:
      """
      Called when check_voice_drift returns violations.
      Makes ONE correction Claude call:
        system = prepend_voice_anchor(ctx,
          "Rewrite this message removing these exact phrases: {violations}.
           Keep the same hook and structure. Keep under {target_length} chars.
           Return only the rewritten message text.")
        prompt = draft
      Returns the corrected draft string.
      Called MAXIMUM ONCE per draft — if violations persist after one fix,
      log a warning and return the fixed draft anyway.
      """

  In draft_message(), after receiving result from ask_json():
    For draft_a:
      violations = check_voice_drift(result["draft_a"])
      if violations:
          result["draft_a"] = fix_drift(result["draft_a"], violations, ctx)
          result["drift_detected"] = True
          result["violations_found"] = violations
    For draft_b (same pattern):
      violations = check_voice_drift(result["draft_b"])
      if violations:
          result["draft_b"] = fix_drift(result["draft_b"], violations, ctx)

  Add "drift_detected": False and "violations_found": [] as defaults in result dict.
