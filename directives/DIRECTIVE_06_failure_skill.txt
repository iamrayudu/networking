DIRECTIVE 06 — FAILURE SKILL
==============================
Problem being solved: when something goes wrong — LinkedIn blocks,
empty search results, missing profile data, Claude returns bad JSON,
Selenium times out — agents have no instruction for what to do.
They either crash silently, loop forever, or pass garbage to the next step.
Sudheer finds out when he looks at the dashboard and sees nothing happened.

Fix: a failure decision tree for every failure type. Each failure has
a defined response: retry once / skip and continue / pause and ask /
stop the agent. Nothing fails silently. Everything is logged and surfaced.

Do NOT rebuild any existing file. Add the failure handler layer.

  ── HANDLE FUNCTION ──
  def handle_failure(failure_type: str, agent, ctx: dict,
                     detail: str = "", **kwargs) -> dict:
      """
      Main failure handler. Called whenever an error occurs.

      Args:
        failure_type: key from FAILURE_POLICY
        agent:        the LinkedInAgent instance (for emit, stop_flag, pause_flag)
        ctx:          current context object
        detail:       optional extra detail string for the message
        **kwargs:     template vars for the message string (e.g. company, role)

      Returns: {"action": str, "should_continue": bool, "ctx": dict}
        should_continue: True = caller continues to next step
                         False = caller must check action and respond

      Steps:
        1. Look up policy = FAILURE_POLICY.get(failure_type)
           If not found: treat as severity=HIGH, action=PAUSE_AND_ASK
        2. Log to agent_log via agent.log():
           action = "FAILURE", detail = f"{failure_type}: {detail}", status = policy["severity"]
        3. If message is not None: format with kwargs, emit AGENT_MESSAGE
        4. Apply action:
           SKIP:          return {"action": "SKIP", "should_continue": True, "ctx": ctx}
           MARK_AND_SKIP: call mark_contacted(kwargs.get("linkedin_url",""), ctx) if url given
                          return {"action": "MARK_AND_SKIP", "should_continue": True, "ctx": ctx}
           RETRY_ONCE:    return {"action": "RETRY", "should_continue": False, "ctx": ctx}
           PAUSE_AND_ASK: agent.pause_flag.set()
                          emit AGENT_STATUS: status="waiting", current_action="paused — failure"
                          return {"action": "PAUSE", "should_continue": False, "ctx": ctx}
           STOP_AGENT:    agent.stop_flag.set()
                          return {"action": "STOP", "should_continue": False, "ctx": ctx}
           STOP_ALL:      emit ERROR with error_type="CAPTCHA_DETECTED" (triggers pool.pause_all)
                          agent.stop_flag.set()
                          return {"action": "STOP_ALL", "should_continue": False, "ctx": ctx}
      """

  ── RETRY WRAPPER ──
  def with_retry(fn, failure_type: str, agent, ctx: dict,
                 max_retries: int = 1, retry_delay: float = 3.0, **kwargs):
      """
      Wraps a callable with one retry on failure.
      On first failure: waits retry_delay seconds, tries once more.
      On second failure: calls handle_failure() with action = SKIP.

      Usage:
        result = with_retry(
            lambda: ask_json(prompt, system=system),
            failure_type="claude_json_parse_error",
            agent=self, ctx=ctx
        )
      Returns the function result, or None if all retries failed.
      """
