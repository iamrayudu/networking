DIRECTIVE 05 — OUTPUT STANDARDS PER STEP
==========================================
Problem being solved: each tool call hands off data to the next step
with no guaranteed shape. A missing field in profile extraction causes
a KeyError in scoring. A None score causes a crash in message drafting.
An incomplete enrichment produces a story with no fit indicators.
Errors propagate silently until they cause garbage output or a crash.

Fix: define an exact output schema for every step. Validate before handoff.
If validation fails: fill with safe defaults, log the gap, continue.
Never crash on missing data — but always know when data is incomplete.

Do NOT rebuild any existing file. Add the schema and validation layer.

      "team_info":             (str,  "Not found.", False),
      "tech_stack":            (list, [],           False),
      "role_signals":          (list, [],           True),
      "culture_signals":       (list, [],           False),
      "linkedin_company_url":  (str,  "not found",  False),
      "fit_indicators":        (list, [],           True),
      "contact_targets":       (list, [],           True),
      "enrichment_confidence": (str,  "LOW",        False),
  }

  ── SCHEMA 5: story output (parsed from .md file) ──
  STORY_SCHEMA = {
      "headline":       (str,  "",  True),
      "key_strengths":  (list, [],  True),
      "talking_points": (list, [],  True),
      "gap_framing":    (str,  "No significant gaps identified.", False),
      "outreach_tone":  (str,  "",  False),
  }

  ── VALIDATE FUNCTION ──
  def validate(data: dict, schema: dict, step_name: str) -> dict:
      """
      Validates data against schema. Returns a clean dict.

      For each field in schema:
        - If field missing from data and required=True:
            log WARNING: f"[{step_name}] Required field '{field}' missing — using default"
            set data[field] = default
        - If field missing from data and required=False:
            set data[field] = default silently
        - If field present but wrong type:
            log WARNING: f"[{step_name}] Field '{field}' wrong type — using default"
            set data[field] = default
        - If field present and correct type: keep as-is

      Special handling:
        - For list fields: if value is None, replace with []
        - For str fields: if value is None, replace with ""
        - For int fields: if value is None, replace with 0
        - For bool fields: if value is None, replace with False

      Returns the cleaned dict (never raises).
      """

  ── SCHEMA LOOKUP ──
  SCHEMAS = {
      "profile":    PROFILE_SCHEMA,
      "score":      SCORE_SCHEMA,
      "draft":      DRAFT_SCHEMA,
      "enrichment": ENRICHMENT_SCHEMA,
      "story":      STORY_SCHEMA,
  }

  def validate_step(data: dict, step: str) -> dict:
      """Convenience wrapper. step must be a key in SCHEMAS."""
      if step not in SCHEMAS:
          raise ValueError(f"Unknown step: {step}. Valid: {list(SCHEMAS.keys())}")
      return validate(data, SCHEMAS[step], step)

CHANGE 3 — ADD CHAR COUNT GUARD ON DRAFTS
═══════════════════════════════════════════════
File to modify: backend/agents/message_agent.py

Tell Claude Code:
  After validate_step on the draft result, add a hard trim:

  LINKEDIN_CHAR_LIMIT = 300

  def trim_to_limit(draft: str, limit: int = LINKEDIN_CHAR_LIMIT) -> str:
      """
      If draft exceeds limit: trim at the last word boundary before limit.
      Never cut mid-word. Never cut mid-sentence if avoidable.
      Returns trimmed string.
      """
      if len(draft) <= limit:
          return draft
      trimmed = draft[:limit]
      last_space = trimmed.rfind(" ")
      if last_space > limit * 0.8:   # only trim at word boundary if not too far back
          trimmed = trimmed[:last_space]
      return trimmed.rstrip(" .,")

  After validate_step in draft_message():
    result["draft_a"] = trim_to_limit(result["draft_a"])
    result["draft_b"] = trim_to_limit(result["draft_b"])
    result["char_count_a"] = len(result["draft_a"])
    result["char_count_b"] = len(result["draft_b"])
